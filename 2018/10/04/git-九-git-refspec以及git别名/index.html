第二种方式：

有些时候我们在本地的分支叫dev，而在远程分支的名字叫dev2，那么我们怎么处理呢？
写法：
git push –set-upstream origin dev:dev2

在推送的时候使用push的全写：git push origin dev:dev2，即显式指定远程分支的名字。

重命名远程分支：
只有一种方式，就是先删除远程分支，然后在本地重命名再提交到远程分支，即先删除后新增。
